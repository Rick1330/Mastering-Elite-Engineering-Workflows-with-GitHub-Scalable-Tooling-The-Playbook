# AI-Assisted Workflows: Augmenting Engineering with LLMs

<Tip>
**TL;DR**: Integrate AI throughout your engineering workflow for PR summaries, automated code reviews, intelligent issue triage, and documentation generation. This guide provides production-ready GitHub Actions for AI-native workflows.
</Tip>

## Introduction

Artificial Intelligence, particularly Large Language Models (LLMs), has transformed from a novelty to an essential tool for elite engineering teams. When integrated thoughtfully into workflows, AI can:
- Reduce manual overhead by 30-50%
- Catch issues before human review
- Accelerate onboarding for new team members
- Provide consistent, 24/7 automation

This guide covers battle-tested patterns for integrating AI into your GitHub workflows.

## Automated PR Summaries

### PR Summary Generator Action

```yaml
# .github/actions/generate-pr-summary/action.yml
name: 'Generate PR Summary'
description: 'Use AI to generate comprehensive PR descriptions'
author: 'Elite Engineering Team'

inputs:
  openai-api-key:
    description: 'OpenAI API key'
    required: true
  model:
    description: 'LLM model to use'
    required: false
    default: 'gpt-4-turbo-preview'
  max-diff-size:
    description: 'Maximum diff size in lines to process'
    required: false
    default: '1000'

outputs:
  summary:
    description: 'Generated PR summary'
    value: ${{ steps.generate.outputs.summary }}
  suggested-reviewers:
    description: 'Suggested reviewers based on changes'
    value: ${{ steps.generate.outputs.reviewers }}

runs:
  using: 'composite'
  steps:
    - name: Generate PR Summary with AI
      id: generate
      shell: bash
      env:
        OPENAI_API_KEY: ${{ inputs.openai-api-key }}
        PR_NUMBER: ${{ github.event.pull_request.number }}
        REPO: ${{ github.repository }}
      run: |
        # Fetch PR diff
        DIFF=$(gh pr diff "$PR_NUMBER" --repo "$REPO" | head -n ${{ inputs.max-diff-size }})
        
        # Get PR title and existing description
        PR_TITLE=$(gh pr view "$PR_NUMBER" --json title -q .title)
        PR_BODY=$(gh pr view "$PR_NUMBER" --json body -q .body)
        
        # Create prompt for LLM
        PROMPT=$(cat <<EOF
        You are an expert software engineer reviewing a pull request.
        
        PR Title: $PR_TITLE
        
        Original Description:
        $PR_BODY
        
        Changes (diff):
        $DIFF
        
        Generate a comprehensive PR summary with:
        1. **Overview**: One-sentence summary of what this PR does
        2. **Key Changes**: Bullet list of major changes (group by component)
        3. **Testing Done**: What tests were added/modified
        4. **Potential Risks**: Any concerns or edge cases to consider
        5. **Suggested Reviewers**: Based on changed files, suggest 2-3 reviewers
        
        Format as Markdown. Be concise but thorough.
        EOF
        )
        
        # Call OpenAI API
        RESPONSE=$(curl -s https://api.openai.com/v1/chat/completions \
          -H "Content-Type: application/json" \
          -H "Authorization: Bearer $OPENAI_API_KEY" \
          -d "{
            \"model\": \"${{ inputs.model }}\",
            \"messages\": [
              {\"role\": \"system\", \"content\": \"You are a helpful assistant that generates PR summaries for software engineers.\"},
              {\"role\": \"user\", \"content\": \"$PROMPT\"}
            ],
            \"max_tokens\": 1000,
            \"temperature\": 0.3
          }")
        
        SUMMARY=$(echo "$RESPONSE" | jq -r '.choices[0].message.content')
        
        echo "summary<<EOF" >> $GITHUB_OUTPUT
        echo "$SUMMARY" >> $GITHUB_OUTPUT
        echo "EOF" >> $GITHUB_OUTPUT
        
        # Extract suggested reviewers (simple pattern matching)
        REVIEWERS=$(echo "$SUMMARY" | grep -oP '@\w+' | head -3 | tr '\n' ',' | sed 's/,$//')
        echo "reviewers=$REVIEWERS" >> $GITHUB_OUTPUT

    - name: Post Summary as Comment
      shell: bash
      env:
        SUMMARY: ${{ steps.generate.outputs.summary }}
      run: |
        gh pr comment "$PR_NUMBER" --repo "$REPO" --body "
        ## 🤖 AI-Generated PR Summary
        
        $SUMMARY
        
        ---
        *Generated by AI. Please verify accuracy.*
        "
```

### Workflow Integration

```yaml
# .github/workflows/pr-summary.yml
name: AI PR Summary

on:
  pull_request:
    types: [opened, synchronize, reopened]
    branches: [main, develop]

jobs:
  generate-summary:
    runs-on: ubuntu-latest
    timeout-minutes: 5
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Generate PR Summary
        uses: ./.github/actions/generate-pr-summary
        with:
          openai-api-key: ${{ secrets.OPENAI_API_KEY }}
          model: gpt-4-turbo-preview
      
      - name: Suggest Reviewers
        if: github.event.pull_request.requested_reviewers == 0
        run: |
          REVIEWERS="${{ steps.generate.outputs.suggested-reviewers }}"
          if [ -n "$REVIEWERS" ]; then
            echo "Suggested reviewers: $REVIEWERS"
            # Optionally auto-request reviewers here
          fi
```

## AI Code Review Assistant

### Automated Review Bot

```yaml
# .github/actions/ai-code-review/action.yml
name: 'AI Code Review'
description: 'Perform preliminary code review using AI'
author: 'Elite Engineering Team'

inputs:
  openai-api-key:
    description: 'OpenAI API key'
    required: true
  focus-areas:
    description: 'Comma-separated list of focus areas (security,performance,readability,tests)'
    required: false
    default: 'security,performance,readability'
  min-confidence:
    description: 'Minimum confidence threshold for suggestions (0-1)'
    required: false
    default: '0.7'

outputs:
  issues-found:
    description: 'Number of issues identified'
    value: ${{ steps.review.outputs.issues_count }}
  has-critical:
    description: 'Whether critical issues were found'
    value: ${{ steps.review.outputs.has_critical }}

runs:
  using: 'composite'
  steps:
    - name: Perform AI Code Review
      id: review
      shell: bash
      env:
        OPENAI_API_KEY: ${{ inputs.openai-api-key }}
        PR_NUMBER: ${{ github.event.pull_request.number }}
      run: |
        # Get changed files
        CHANGED_FILES=$(gh pr view "$PR_NUMBER" --json files -q '.files[].path')
        
        # Build review context
        CONTEXT=""
        for FILE in $CHANGED_FILES; do
          if [[ -f "$FILE" ]]; then
            CONTEXT+="File: $FILE\n"
            CONTEXT+="$(head -100 "$FILE")\n\n"
          fi
        done
        
        DIFF=$(gh pr diff "$PR_NUMBER")
        
        # Create focused review prompt
        PROMPT=$(cat <<EOF
        You are a senior software engineer performing a code review. Focus on these areas: ${{ inputs.focus-areas }}.
        
        Pull Request Diff:
        $DIFF
        
        Changed Files Context:
        $CONTEXT
        
        Identify issues and provide feedback. For each issue:
        1. **Severity**: CRITICAL, HIGH, MEDIUM, or LOW
        2. **Location**: File and line number
        3. **Issue**: Clear description of the problem
        4. **Suggestion**: Specific code change or improvement
        5. **Confidence**: Your confidence level (0-1)
        
        Only report issues with confidence >= ${{ inputs.min-confidence }}.
        Prioritize security vulnerabilities, performance issues, and bugs.
        Skip style issues that would be caught by linters.
        
        Format as a Markdown table with columns: Severity | Location | Issue | Suggestion
        EOF
        )
        
        # Call LLM API
        RESPONSE=$(curl -s https://api.openai.com/v1/chat/completions \
          -H "Content-Type: application/json" \
          -H "Authorization: Bearer $OPENAI_API_KEY" \
          -d "{
            \"model\": \"gpt-4-turbo-preview\",
            \"messages\": [
              {\"role\": \"system\", \"content\": \"You are an expert code reviewer. Be constructive and specific.\"},
              {\"role\": \"user\", \"content\": \"$PROMPT\"}
            ],
            \"max_tokens\": 2000,
            \"temperature\": 0.2
          }")
        
        FEEDBACK=$(echo "$RESPONSE" | jq -r '.choices[0].message.content')
        
        # Count issues
        ISSUES_COUNT=$(echo "$FEEDBACK" | grep -c "CRITICAL\|HIGH\|MEDIUM" || echo "0")
        HAS_CRITICAL=$(echo "$FEEDBACK" | grep -q "CRITICAL" && echo "true" || echo "false")
        
        echo "issues_count=$ISSUES_COUNT" >> $GITHUB_OUTPUT
        echo "has_critical=$HAS_CRITICAL" >> $GITHUB_OUTPUT
        
        # Save feedback for posting
        echo "$FEEDBACK" > /tmp/ai-review.md

    - name: Post Review Feedback
      shell: bash
      run: |
        if [ -f /tmp/ai-review.md ]; then
          FEEDBACK=$(cat /tmp/ai-review.md)
          
          if [[ "${{ steps.review.outputs.issues_count }}" -gt 0 ]]; then
            gh pr comment "$PR_NUMBER" --body "
            ## 🤖 AI Code Review Feedback
            
            I've analyzed this PR and found **${{ steps.review.outputs.issues_count }}** potential issues to address:
            
            $FEEDBACK
            
            ---
            *Note: This is an automated review. Please use your judgment when applying these suggestions.*
            "
            
            if [[ "${{ steps.review.outputs.has_critical }}" == "true" ]]; then
              echo "⚠️ CRITICAL issues found - alerting reviewers"
            fi
          else
            echo "✅ No significant issues found by AI review"
          fi
        fi
```

## Intelligent Issue Triage

### Auto-Triage Action

```yaml
# .github/actions/auto-triage-issue/action.yml
name: 'Auto-Triage Issue'
description: 'Automatically categorize and assign issues using AI'
author: 'Elite Engineering Team'

inputs:
  openai-api-key:
    description: 'OpenAI API key'
    required: true
  
outputs:
  category:
    description: 'Detected issue category'
    value: ${{ steps.triage.outputs.category }}
  priority:
    description: 'Suggested priority'
    value: ${{ steps.triage.outputs.priority }}
  assigned-team:
    description: 'Suggested team assignment'
    value: ${{ steps.triage.outputs.team }}

runs:
  using: 'composite'
  steps:
    - name: Analyze and Triage Issue
      id: triage
      shell: bash
      env:
        OPENAI_API_KEY: ${{ inputs.openai-api-key }}
        ISSUE_NUMBER: ${{ github.event.issue.number }}
      run: |
        # Get issue details
        ISSUE=$(gh issue view "$ISSUE_NUMBER" --json title,body,labels,comments)
        TITLE=$(echo "$ISSUE" | jq -r '.title')
        BODY=$(echo "$ISSUE" | jq -r '.body')
        COMMENTS=$(echo "$ISSUE" | jq -r '.comments[].body' | head -5)
        
        # Create classification prompt
        PROMPT=$(cat <<EOF
        Analyze this GitHub issue and classify it:
        
        Title: $TITLE
        
        Description:
        $BODY
        
        Existing Comments:
        $COMMENTS
        
        Classify into exactly ONE category from this list:
        - bug
        - feature-request
        - documentation
        - performance
        - security
        - question
        - enhancement
        
        Assign priority: critical, high, medium, or low
        
        Suggest team assignment: frontend, backend, devops, platform, security, docs
        
        Also extract any mentioned:
        - Affected components/services
        - Error messages
        - Steps to reproduce (if bug)
        
        Respond in JSON format:
        {
          "category": "...",
          "priority": "...",
          "team": "...",
          "components": ["..."],
          "confidence": 0.xx,
          "summary": "One sentence summary"
        }
        EOF
        )
        
        # Call LLM API
        RESPONSE=$(curl -s https://api.openai.com/v1/chat/completions \
          -H "Content-Type: application/json" \
          -H "Authorization: Bearer $OPENAI_API_KEY" \
          -d "{
            \"model\": \"gpt-4-turbo-preview\",
            \"messages\": [
              {\"role\": \"system\", \"content\": \"You are an issue triage assistant. Be precise and accurate.\"},
              {\"role\": \"user\", \"content\": \"$PROMPT\"}
            ],
            \"max_tokens\": 500,
            \"temperature\": 0.1,
            \"response_format\": { \"type\": \"json_object\" }
          }")
        
        # Parse response
        CATEGORY=$(echo "$RESPONSE" | jq -r '.choices[0].message.category')
        PRIORITY=$(echo "$RESPONSE" | jq -r '.choices[0].message.priority')
        TEAM=$(echo "$RESPONSE" | jq -r '.choices[0].message.team')
        CONFIDENCE=$(echo "$RESPONSE" | jq -r '.choices[0].message.confidence')
        SUMMARY=$(echo "$RESPONSE" | jq -r '.choices[0].message.summary')
        
        echo "category=$CATEGORY" >> $GITHUB_OUTPUT
        echo "priority=$PRIORITY" >> $GITHUB_OUTPUT
        echo "team=$TEAM" >> $GITHUB_OUTPUT
        
        # Apply labels
        gh issue edit "$ISSUE_NUMBER" --add-label "$CATEGORY,$PRIORITY"
        
        # Add triage comment
        gh issue comment "$ISSUE_NUMBER" --body "
        ### 🤖 Automated Triage
        
        **Category:** \`$CATEGORY\`  
        **Priority:** \`$PRIORITY\`  
        **Suggested Team:** \`$TEAM\`  
        **Confidence:** ${CONFIDENCE}%
        
        **Summary:** $SUMMARY
        
        ---
        *Triage performed automatically. Please adjust if incorrect.*
        "
        
        # Optionally assign to team queue
        # gh issue edit "$ISSUE_NUMBER" --add-assignee "$TEAM-bot"
```

## Documentation Generation

### Auto-Doc Generator

```yaml
# .github/actions/generate-docs/action.yml
name: 'Generate Documentation'
description: 'Auto-generate documentation from code changes'
author: 'Elite Engineering Team'

inputs:
  openai-api-key:
    description: 'OpenAI API key'
    required: true
  doc-type:
    description: 'Type of docs to generate (api,readme,changelog)'
    required: false
    default: 'api'

runs:
  using: 'composite'
  steps:
    - name: Generate API Documentation
      if: inputs.doc-type == 'api'
      shell: bash
      env:
        OPENAI_API_KEY: ${{ inputs.openai-api-key }}
      run: |
        # Find changed/added files
        CHANGED=$(git diff --name-only HEAD~1 HEAD | grep -E '\.(ts|js|py|go)$' || true)
        
        if [ -z "$CHANGED" ]; then
          echo "No relevant code changes found"
          exit 0
        fi
        
        for FILE in $CHANGED; do
          if [[ -f "$FILE" ]]; then
            CONTENT=$(cat "$FILE")
            
            PROMPT="Generate API documentation for this code. Include:
            - Purpose and functionality
            - Parameters and return values
            - Usage examples
            - Any important notes or caveats
            
            Code:
            $CONTENT
            
            Format as Markdown suitable for inclusion in API docs."
            
            DOCS=$(curl -s https://api.openai.com/v1/chat/completions \
              -H "Content-Type: application/json" \
              -H "Authorization: Bearer $OPENAI_API_KEY" \
              -d "{
                \"model\": \"gpt-4-turbo-preview\",
                \"messages\": [{\"role\": \"user\", \"content\": \"$PROMPT\"}],
                \"max_tokens\": 1500
              }" | jq -r '.choices[0].message.content')
            
            # Append to docs file
            echo -e "\n\n## $FILE\n" >> docs/api-auto-generated.md
            echo "$DOCS" >> docs/api-auto-generated.md
          fi
        done
        
        # Commit generated docs
        if git diff --quiet docs/api-auto-generated.md; then
          echo "No new documentation generated"
        else
          git add docs/api-auto-generated.md
          git commit -m "docs: Auto-generate API documentation [skip ci]"
          git push
        fi
```

## Implementation Best Practices

### Cost Management

```yaml
# Budget tracking for AI API calls
name: AI Workflow Budget Tracker

on:
  workflow_run:
    workflows: ["*AI*"]
    types: [completed]

jobs:
  track-usage:
    runs-on: ubuntu-latest
    steps:
      - name: Calculate API Costs
        run: |
          # Track token usage from workflow logs
          # Alert if exceeding monthly budget
          MONTHLY_USAGE=$(grep -o 'tokens: [0-9]*' logs/*.txt | awk '{sum+=$2} END {print sum}')
          BUDGET=100000  # tokens per month
          
          if [ "$MONTHLY_USAGE" -gt "$BUDGET" ]; then
            echo "⚠️ AI API usage exceeded budget!"
            # Send alert
          fi
```

## Ethical Considerations

<Warning>
**Important Guidelines**:
1. Always disclose AI-generated content
2. Never fully automate security decisions
3. Maintain human oversight for critical changes
4. Respect privacy - don't send sensitive code to external APIs
5. Use AI as augmentation, not replacement for human judgment
</Warning>
